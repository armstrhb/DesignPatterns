Design Patterns

Common Software Patterns, laid out and explained simply.

If you can't explain it simply, you don't understand it well enough.

Albert Einstein Richard Reynman Lord Rutherford of Nelson Someone smart

Sometimes there is a better way to do things.

All developers have been at this stage at some point in their career. Some are smart, and begin to yearn for this knowledge early. Some learn these in class, not understanding their importance outside of their assignments. And some, like myself, learn these through doing things the hard way first.

Design Patterns exist for common problems found in software development. Simple knowledge of the existance of these is a start for the day we live in, when the power of the internet (read google) is at the fingertips of every developer.

Having in-head knowledge of these structures, and being able to apply them directly without reference, is a big step to removing yourself from code monkey status.

I've set out to learn the common design patterns out there in the world. Not just to learn them, but to be able to lay out their ideas, their goals, and the thoughts behind them simply.

This is a personal project. But I'll keep what I learn for you to benefit from.

Each Pattern will be laid out and explained. For each, I'll do the following.

An (Extremely) Simple Description

As a father of two little boys, I've come to appreciate the skill it takes to explain complex concepts to a small child.

Each Design Pattern will have a simple description, broken down to a level that a five year old would understand.

Further Detail of the Pattern

More detail on the problem that this pattern solves, and specifics on how the pattern solves it.

Code Examples

What's a website about code development without code?

Examples of the design pattern will be laid out in simple code samples.
